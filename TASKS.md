### Instructions

1. Update your local main to reflect the remote main.

2. Run the local app again to see the changes that have been made. This will inform you on what needs to be worked on.

3. Make a branch off the main local branch, and solve your task there. NEVER WORK ON THE MAIN BRANCH. It is the source of truth for any project. Only work on branches created off the main branch

4. After solving your task, stage, commit with a appropriate commit message that reflects what you worked on, push the branch to remote (you should research how to do this).

5. Raise a Pull Request (PR) with the appropraite and more elaborate note on what you solved in your task, so the Project lead (Terdoo) can merge into remote main. When you raise PRs, you have to be detailed on what the problem was, and what you did to solve it.

6. Your PR may be merged or rejected by the Project Lead, in the case of a rejection, an appropriate note will be left by the PL.

7. After the PR has been merged, you are required to update your local Main again. You always have to do this.
